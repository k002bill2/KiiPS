# 반합리화 규칙 (Anti-Rationalization)

> Claude Forge Golden Principle #7(Ralph Loop 감지) + #5(수술적 변경) + #11(반합리화)를 KiiPS 환경에 적응

## 금지 표현 테이블

| 합리화 | 올바른 행동 |
|--------|------------|
| "여기 온 김에 정리하자" | 요청된 것만 수정. 정리는 별도 요청입니다 |
| "확장성을 위해 추상화" | 지금 필요한 것만. 반복이 3회 이상일 때 추상화 |
| "더 나은 방법 발견했으니 변경" | 사용자 승인 후 변경 |
| "이 패턴 전체에 적용하자" | 요청 범위만 수정 |
| "나중에 문제될 수 있으니 미리" | 현재 문제만 해결 |
| "이미 잘 동작하니 추가 검증 불필요" | 증거 없는 주장은 거짓입니다 |
| "테스트 작성은 나중에" | 나중에 작성한 테스트는 happy path만 커버합니다 |
| "간단한 수정이라 빌드 불필요" | 간단한 수정이 빌드를 깨뜨립니다 |

## HARD-GATE 규칙

### 다중 파일 변경 게이트
- **3개 이상 파일 변경 예정 시** → 사용자에게 변경 파일 목록 제시 후 승인 요청
- 승인 없이 진행 금지

### 고위험 파일 게이트
다음 파일 변경 시 **반드시** 사용자 승인:
- `KiiPS-COMMON/` 모듈 내 모든 파일
- `KiiPS-UTILS/` 모듈 내 모든 파일
- `pom.xml` (모든 모듈)
- `app-*.properties` (환경 설정)
- MyBatis mapper XML (SQL 변경)

## Ralph Loop 감지

### 파일 반복 편집 감지
- **동일 파일 3회 이상 Edit** → 즉시 중단
- 진단 모드 전환: 왜 반복 편집이 필요한지 근본 원인 분석
- 사용자에게 상황 보고 후 방향 재설정

### 빌드 실패 반복 감지
- **빌드 실패 3회 연속** → 즉시 중단
- 근본 원인 분석 모드 전환
- 자동 롤백 프로토콜 실행 (아래 참조)

### 에러 악순환 감지
- 수정 A → 새 에러 B → 수정 B → 새 에러 C → **중단**
- 최초 변경 지점으로 되돌리기
- 접근 방식 재설계

### 자동 롤백 프로토콜

Ralph Loop 감지 시 다음 절차를 **순서대로** 실행:

```
1. HALT    — 즉시 모든 편집 중단
2. REPORT  — 사용자에게 상황 보고
   - 반복 편집된 파일 목록
   - 각 편집의 의도와 결과
   - 빌드/에러 로그 요약
3. REVERT  — 사용자 승인 후 변경사항 되돌리기
   - SVN: `svn revert <변경된 파일들>`
   - 되돌릴 파일 목록을 사용자에게 먼저 제시
   - 사용자가 "되돌려" 또는 승인 시 즉시 실행
4. DIAGNOSE — 근본 원인 분석 (/diagnose 활용)
   - 왜 반복 실패했는지 가설 제시
   - 대안 접근법 제안
5. RESTART — 새로운 접근법으로 재시작 (사용자 승인 후)
```

**자동 롤백 트리거 조건:**

| 조건 | 임계값 | 롤백 대상 |
|------|--------|----------|
| 동일 파일 반복 편집 | 3회 | 해당 파일 |
| 연속 빌드 실패 | 3회 | 이번 세션 변경 파일 전체 |
| 에러 악순환 (A→B→C) | 3단계 | 최초 변경 시점까지 |

**금지사항:**
- 사용자 승인 없이 `svn revert` 실행 금지
- 되돌리기 범위를 임의로 축소 금지 (제시된 전체 파일 되돌리기)
- 롤백 후 동일한 접근법으로 재시도 금지

## 수술적 변경 원칙

1. **추적 가능성** - 모든 변경 줄은 사용자 요청에 추적 가능해야 함
2. **"Bonus fix" 금지** - 발견한 다른 문제는 보고만, 수정은 별도 요청 시
3. **범위 고정** - 작업 시작 시 합의된 범위를 초과하지 않음
4. **관심사 분리** - 버그 수정 + 리팩토링 + 기능 추가를 하나의 작업에 혼합 금지

## KiiPS 특화 규칙

| 상황 | 금지 행동 | 올바른 행동 |
|------|----------|------------|
| JSP 수정 중 다른 JSP 오류 발견 | 함께 수정 | 보고만 하고 별도 작업으로 |
| SCSS 수정 중 구조 개선 필요 발견 | 리팩토링 병행 | 요청된 스타일만 수정 |
| MyBatis ${} 발견 | 전체 파일 #{} 변환 | 요청 범위 내 변환 + 나머지 보고 |
| Controller 수정 중 Service 개선점 발견 | 함께 수정 | Controller만 수정 + 보고 |
| 빌드 실패 시 의존성 정리 | 캐시 삭제 + 버전 변경 | 에러 메시지 분석 우선 |

## 핵심

```
요청된 것만 수정하세요.
발견한 것은 보고하세요.
승인 없이 확장하지 마세요.
```
