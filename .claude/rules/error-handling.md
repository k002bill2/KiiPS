# Error Handling Rules

> CLAUDE.md에서 분리된 에러 처리 상세 규칙

## 핵심 규칙

1. **근본 원인 우선** - 코드 변경 전 가설을 먼저 제시
2. **캐시 삭제 금지** - 사용자 확인 없이 `.m2`, `node_modules` 등 삭제 금지
3. **악화 시 중단** - 수정이 상황을 악화시키면 즉시 되돌리기
4. **한 번에 하나** - 5개 변경을 동시에 하지 말고, 하나씩 검증
5. **상태 보존** - 디버깅 중 관련 없는 작동 중인 파일 수정 금지

## 디버깅 프로토콜

```
1. 에러 메시지 정확히 읽기
2. 가설 제시 (최소 2개)
3. 가장 가능성 높은 가설부터 검증
4. 한 번에 하나의 변경만 적용
5. 변경 후 즉시 검증
6. 악화 시 즉시 되돌리기
```

## 금지 행동

- 사용자 확인 없이 캐시/빌드 결과물 삭제
- 에러 원인 파악 전 "일단 시도" 식 변경
- 디버깅 중 "여기도 고치자" 식 확장
- 작동 중인 코드를 디버깅 과정에서 수정
