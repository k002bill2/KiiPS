# Code Quality Rubric

KiiPS 백엔드 코드 품질 평가 기준

## 평가 항목

### 1. 보안 (35%)
- [ ] SQL Injection 방어 (PreparedStatement 사용)
- [ ] XSS 방어 적용
- [ ] 자격 증명 하드코딩 없음
- [ ] 입력값 검증

### 2. 에러 처리 (25%)
- [ ] try-catch 적절히 사용
- [ ] 의미 있는 에러 메시지
- [ ] 로깅 포함
- [ ] 리소스 정리 (finally/try-with-resources)

### 3. 코드 구조 (20%)
- [ ] 단일 책임 원칙
- [ ] 적절한 메소드 분리
- [ ] 중복 코드 없음
- [ ] 명확한 네이밍

### 4. 성능 (10%)
- [ ] N+1 쿼리 없음
- [ ] 적절한 캐싱
- [ ] 불필요한 루프 없음

### 5. 유지보수성 (10%)
- [ ] JavaDoc 주석
- [ ] 매직 넘버 없음 (상수 사용)
- [ ] 테스트 가능한 구조

## 점수 기준

| 점수 | 설명 |
|------|------|
| 90-100 | 우수: 프로덕션 레디, 보안 완벽 |
| 70-89 | 양호: 대부분 안전, 개선 여지 있음 |
| 50-69 | 보통: 기본 동작, 보안 검토 필요 |
| 0-49 | 미흡: 보안 취약점 또는 심각한 결함 |
