# KiiPS System Integrity Test Results

This directory contains test reports generated by the KiiPS system integrity tests.

## Report Format

Reports are generated in Markdown format with the filename pattern: `report-YYYY-MM-DD.md`

Each report contains:

1. **Summary**: Overall test results and pass rate
2. **Detailed Results**: Individual test suite results
3. **Recommendations**: Actions to take based on test results

## Test Suites

### 1. Configuration Tests (`test-config.js`)
Validates system configuration files:
- `.claudecode.json` structure and hooks
- `skill-rules.json` rule definitions
- Hook matcher regex patterns
- File references and dependencies

### 2. Skills Tests (`test-skills.js`)
Verifies KiiPS skills integrity:
- SKILL.md file existence for each skill
- Required sections (YAML frontmatter, Purpose, Examples, Related Skills)
- Skill structure completeness

### 3. Hook Activation Tests (`test-hook-activation.js`)
Simulates skill activation scenarios:
- Keyword matching accuracy
- Intent pattern matching correctness
- Expected skill activation for common requests

### 4. ACE Framework Tests (`test-ace-framework.js`)
Checks ACE Framework files and configuration:
- Required ACE files existence
- JSON validity for configuration files
- ACE Framework enabled in .claudecode.json
- Layer references correctness

## Running Tests

```bash
# From project root
cd tests/
./run-tests.sh
```

Or directly with Node:

```bash
node tests/run-all-tests.js
```

## Interpreting Results

### Pass Rate
- **100%**: Perfect! All systems operational
- **90-99%**: Mostly healthy, minor issues to address
- **70-89%**: Several issues found, review recommendations
- **<70%**: Critical issues, immediate attention required

### Exit Code
- **0**: All tests passed
- **1**: One or more tests failed

## Troubleshooting Failed Tests

### Configuration Test Failures
- Check `.claudecode.json` syntax (must be valid JSON)
- Verify all hook types are defined
- Ensure hook matchers use valid regex (not Glob patterns)
- Confirm all referenced files exist

### Skills Test Failures
- Create missing SKILL.md files
- Add required sections (Purpose, Examples, Related Skills)
- Add YAML frontmatter to skill files

### Hook Activation Test Failures
- Review keyword lists in skill-rules.json
- Check intent patterns (must be valid regex)
- Test activation with actual user prompts

### ACE Framework Test Failures
- Verify ACE files exist in .claude/ace-framework/
- Check ACE Framework enabled in .claudecode.json
- Ensure coordination scripts present
