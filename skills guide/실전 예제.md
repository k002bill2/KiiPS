## 🎯 실제 워크플로우 예제

### 예제 1: 새 React 프로젝트 시작하기

#### Step 1: 프로젝트 초기화 (macOS)
```bash
# 프로젝트 생성 (Next.js 16 최신)
npx create-next-app@latest my-app --typescript --tailwind --app
cd my-app

# Claude Code/.agent 설정 (Antigravity 호환)
mkdir -p .agent/workflows .claude/skills .claude/commands
```

#### Step 2: CLAUDE.md 작성
```markdown
# My App - E-commerce Platform

## Project Overview
- **Purpose**: Online shopping platform
- **Tech Stack**: Next.js 16, TypeScript, Tailwind CSS, Prisma
- **Features**: Product catalog, Cart, Checkout, User auth

## Current Tasks
1. Set up authentication with NextAuth
2. Create product listing page
3. Implement shopping cart
4. Add payment integration

## Code Standards
- Use Server Components by default
- Client Components only when needed
- Proper error boundaries
- Loading states for all async operations
```

#### Step 3: Claude Code 세션 시작
```bash
claude

# 첫 작업 요청
"Using the project context in CLAUDE.md, set up the basic authentication system with NextAuth and create the initial user model with Prisma"

# Sub-agent 생성
/agents
# Create 'auth-specialist' agent for authentication tasks

# Skill 생성
"Use the skill-creator to create a skill for generating Prisma models"
```


```
Claude: I'll investigate this issue. Let me first examine the calculateTotal function...

I found the issue: The tax calculation is being applied after the discount, 
causing negative values that default to $0. Here's the fix...

[코드 수정 제안]

I'll also add tests to prevent this regression...
```

### 예제 4: 코드 리팩토링 프로젝트

#### Large-scale 리팩토링 with Sub-agents

```bash
# 메인 orchestrator 설정
claude

"I need to refactor our legacy JavaScript codebase to TypeScript. 
The project has about 50 files. Create a systematic approach using multiple agents."

# Claude가 자동으로 여러 agents 활용:
# - migration-planner: 마이그레이션 계획 수립
# - type-generator: TypeScript 타입 생성
# - test-updater: 테스트 업데이트
# - validator: 타입 체크 및 검증
```

#### 실행 과정
```markdown
1. **분석 단계**
   - migration-planner가 파일 의존성 분석
   - 마이그레이션 순서 결정

2. **변환 단계**
   - type-generator가 인터페이스 생성
   - 파일별 순차 변환

3. **검증 단계**
   - validator가 타입 체크
   - test-updater가 테스트 수정

4. **완료**
   - 최종 리포트 생성
```

## 🛠️ Skills 활용 실전 예제

### Custom Skill 만들기: PR Description Generator

```markdown
# .claude/skills/pr-generator/SKILL.md
---
name: pr-generator
description: Generate comprehensive PR descriptions from git diff and commit messages
---

# PR Description Generator

## Instructions
1. Analyze git diff
2. Read commit messages
3. Identify changed files
4. Detect breaking changes
5. Generate PR description

## Output Format
\`\`\`markdown
## Summary
[Brief description of changes]

## Changes
- Feature: [New features added]
- Fix: [Bugs fixed]
- Refactor: [Code improvements]

## Breaking Changes
[Any breaking changes]

## Testing
- [ ] Unit tests updated
- [ ] E2E tests passed
- [ ] Manual testing completed

## Screenshots
[If UI changes]

## Related Issues
Closes #[issue-number]
\`\`\`

## Usage
Run: git diff main..HEAD | claude "Generate PR description using pr-generator skill"
```

```bash

"Save the current progress to PROGRESS.md"

# 다음 세션에서
"Continue from PROGRESS.md"
```

### 3. Automated Workflows
```json
// .claudecode.json - 자동 워크플로우
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write(*.tsx)",
        "hooks": [
          {"type": "command", "command": "npx tsc --noEmit"},
          {"type": "command", "command": "npm run lint:fix"}
        ]
      }
    ]
  }
}
```

## 🚨 일반적인 문제 해결

### Case 1: Skill이 동작하지 않을 때
```bash
# 디버깅 단계
1. "List all available skills"
2. "Check if code-reviewer skill is loaded"
3. "Reload skills from .claude/skills"
4. /clear 후 재시도
```

### Case 2: Agent가 잘못된 작업 수행
```bash
# Agent 수정
/agents
# Edit agent description to be more specific
# Limit tool access
# Add constraints in system prompt
```

### Case 3: 컨텍스트 오버플로우
```bash
# 해결 방법
1. /clear로 리셋
2. 작업을 작은 단위로 분할
3. 불필요한 파일 컨텍스트 제거
4. Summary를 만들어 컨텍스트 압축
```

## 🏢 KiiPS 엔터프라이즈 예시

### 예시 1: KiiPS 서비스 빌드 & 배포
```bash
# 사용자: "FD 서비스를 빌드하고 배포해줘"

# Claude (kiips-maven-builder 활성화):
cd KiiPS-HUB/
mvn clean package -pl :KiiPS-FD -am

# Build 성공 후 kiips-service-deployer 활성화:
cd ../KiiPS-FD/
./stop.sh && sleep 2
./start.sh
tail -f logs/log.$(date "+%Y-%m-%d")-0.log
```

### 예시 2: KiiPS API Gateway 테스트
```bash
# 사용자: "API Gateway 라우팅 테스트해줘"

# Claude (kiips-api-tester 활성화):
# 1. JWT 토큰 획득
TOKEN=$(curl -s -X POST http://localhost:8801/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"pwd"}' | jq -r '.token')

# 2. 각 서비스 테스트
curl -H "X-AUTH-TOKEN: $TOKEN" http://localhost:8000/api/funds/list
curl -H "X-AUTH-TOKEN: $TOKEN" http://localhost:8000/api/investments/list
curl -H "X-AUTH-TOKEN: $TOKEN" http://localhost:8000/api/programs/list
```

### 예시 3: KiiPS 로그 모니터링
```bash
# 사용자: "에러 로그 분석해줘"

# Claude (kiips-log-analyzer 활성화):
echo "=== FD 서비스 에러 분석 ==="
grep -i "error\|exception" KiiPS-FD/logs/log.$(date "+%Y-%m-%d")-0.log | tail -20

echo "=== 느린 쿼리 분석 ==="
grep "execution time" KiiPS-FD/logs/log.$(date "+%Y-%m-%d")-0.log \
  | awk '$NF > 1000' | tail -10
```
